       Token Bearer compartilhado entre `pipeelo-onboarding-flow` (sender) e `admin-pipeelo` (receiver) que autentica `POST /api/clients/onboarding/create`.

       - Janela de baixo tráfego (idealmente fora do horário comercial BRT)

       ## Procedimento (~10 minutos com outbox)

       1. **Gerar novo token** localmente:

          node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

    4. Cria `scripts/smoke-webhook-pipeline.mjs` (onboarding-flow):
       ```js
       #!/usr/bin/env node
       /**
        * Smoke E2E do pipeline Phase 2.
        * Cria sessão teste → completa → verifica que outbox marcou delivered.
        *
        * Uso: node scripts/smoke-webhook-pipeline.mjs --env=staging
        *
        * Requer: SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY, ADMIN_URL, ONBOARDING_WEBHOOK_TOKEN
        */
       import { createClient } from '@supabase/supabase-js';

       const env = process.argv.find(a => a.startsWith('--env='))?.split('=')[1] ?? 'staging';
       const supabaseUrl = process.env.SUPABASE_URL;
       const serviceKey = process.env.SUPABASE_SERVICE_ROLE_KEY;
       const adminUrl = process.env.PIPEELO_ADMIN_API_URL ?? 'https://admin.pipeelo.com';
       const webhookToken = process.env.ONBOARDING_WEBHOOK_TOKEN;

       if (!supabaseUrl || !serviceKey || !webhookToken) {
         console.error('Missing env vars');
         process.exit(2);
       }

       const sb = createClient(supabaseUrl, serviceKey);
       const sessionId = `smoke_${Date.now()}`;

       // 1. POST direto ao admin com payload válido (skipping sender — testando boundary)
       const payload = {
         payload_version: 'v1',
         session: {
           id: sessionId,
           empresa_nome: 'SMOKE TEST',
           ceo_email: 'smoke@test.com',
           cnpj: '11222333000181',
           created_at: new Date().toISOString(),
         },
         respostas: { identificacao: { smoke: true } },
       };

       console.log(`[smoke ${env}] POST ${adminUrl}/api/clients/onboarding/create  (sessionId=${sessionId})`);
       const resp = await fetch(`${adminUrl}/api/clients/onboarding/create`, {
         method: 'POST',
         headers: {
           'Content-Type': 'application/json',
           'Authorization': `Bearer ${webhookToken}`,
           'Idempotency-Key': sessionId,
         },
         body: JSON.stringify(payload),
       });
       const body = await resp.text();
       console.log(`[smoke ${env}] response: ${resp.status} ${body.slice(0, 200)}`);

       // 2. Idempotency: chama de novo, deve retornar 200 (skipped)
       const resp2 = await fetch(`${adminUrl}/api/clients/onboarding/create`, {
         method: 'POST',
         headers: {
           'Content-Type': 'application/json',
           'Authorization': `Bearer ${webhookToken}`,
         },
         body: JSON.stringify(payload),
       });
       console.log(`[smoke ${env}] re-entry: ${resp2.status}`);

       // 3. Payload inválido — deve retornar 400 com issues
       const invalidResp = await fetch(`${adminUrl}/api/clients/onboarding/create`, {
         method: 'POST',
         headers: {
           'Content-Type': 'application/json',
           'Authorization': `Bearer ${webhookToken}`,
         },
         body: JSON.stringify({ session: { id: 'x' }, respostas: {} }),
       });
       console.log(`[smoke ${env}] invalid: ${invalidResp.status}`);

       const ok = resp.ok && resp2.ok && invalidResp.status === 400;
       console.log(`[smoke ${env}] ${ok ? 'PASS' : 'FAIL'}`);
       process.exit(ok ? 0 : 1);
       ```

    Importante: este script é **manual** (não automatizado em CI) — roda apenas quando humano valida cutover. Não vai pra suite Vitest.
